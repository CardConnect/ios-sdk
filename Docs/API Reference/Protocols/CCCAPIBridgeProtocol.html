Overview

CCC_authApplePayTransactionWithToken:completion:

Called when an ApplePay transaction is initiated.

- (void)CCC_authApplePayTransactionWithToken:(NSString *)token completion:(void ( ^ ) ( BOOL result , NSError *error ))completion

Parameters

token

The token generated for the ApplePay transaction.

completion

A completion block that must be called to return the result of the transaction.

Discussion

This function should be used to process a transaction with the provided ApplePay token. The result of the transaction should be returned in the completion block to notify the CCCPaymentController of the result.

CCC_getAccounts:

Should retrieve a list of accounts for your customer.

- (void)CCC_getAccounts:(void ( ^ ) ( NSArray<CCCAccount*> *accounts , NSError *error ))completion

Parameters

completion

A completion block that should return an array of CCCAccount objects and an error if one occurred.

Discussion

This function should make a call to your backend that retrieves the accounts associated with your customer and return them in the completion handler. Accounts can be initialized using the initWithData: initializer on CCCAccount. You can pass what is returned by your backend CardConnect profile call right into this. The completion block must be called for the process to continue.

CCC_saveAccountToCustomer:completion:

Should save an account for your customer.

- (void)CCC_saveAccountToCustomer:(CCCAccount *)account completion:(void ( ^ ) ( CCCAccount *account , NSError *error ))completion

Parameters

account

The account to send to your backend for saving.

completion

A completion block that should return the saved account returned or an error if one occurred.

Discussion

This function should make a call to your backend that passes the account object and saves it for the customer. Data for account can be obtained by calling [CCCAccount dataForObject]. The completion block must be called for the process to continue.
