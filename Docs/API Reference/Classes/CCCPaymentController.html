CCCPaymentController Class Reference

Inherits from NSObject
Declared in CCCPaymentController.h

Overview

Used to present a pre-build UI that will call your implementation of CCCAPIBridge and return the selected payment account.

The class must use one of the two initialization classes.

presentPaymentView

Presents the payment controller as a modal from your root view.

- (void)presentPaymentView

Discussion

If the payment controller was not initialized with a rootView, api bridge and delegate, this will do nothing. If the root view is already presenting, this will do nothing.
